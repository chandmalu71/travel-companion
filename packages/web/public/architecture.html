4. Key Data Flows

Email booking extraction, notification scheduling, and offline sync flows.

4a. Email Booking Extraction

sequenceDiagram participant User participant Gmail/Outlook participant Webhook participant SQS participant EmailParser participant Dedup participant DB User->>Gmail/Outlook: Receives booking email Gmail/Outlook->>Webhook: Push notification Webhook->>SQS: Enqueue message SQS->>EmailParser: Process email EmailParser->>EmailParser: Extract fields (Comprehend) EmailParser->>Dedup: Check for duplicates alt Not duplicate Dedup->>DB: Create booking DB-->>User: Notify new booking else Duplicate Dedup-->>EmailParser: Discard end

4b. Notification Scheduling

sequenceDiagram participant Booking participant NotifScheduler participant DB participant Worker participant FCM/APNs Booking->>NotifScheduler: Booking created/updated NotifScheduler->>NotifScheduler: Calculate fire_at (event - offset) NotifScheduler->>DB: Store scheduled_notification Worker->>DB: Poll due notifications (every 60s) DB-->>Worker: Return due items Worker->>FCM/APNs: Deliver notification Worker->>DB: Mark as sent

4c. Offline Sync Protocol

sequenceDiagram participant Client participant SyncEngine participant Server participant DB Client->>Client: Queue local changes while offline Client->>SyncEngine: Connectivity restored SyncEngine->>Server: POST /api/sync {lastSyncTimestamp, localChanges} Server->>DB: Apply non-conflicting changes Server->>Server: Detect conflicts (last-write-wins) Server-->>SyncEngine: {serverChanges, conflicts, newTimestamp} SyncEngine->>Client: Apply server changes SyncEngine->>Client: Notify conflicts to user
